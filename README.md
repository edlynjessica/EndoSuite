# EndoSuite: Clinical Dashboard UI/UX Prototype

This repository houses a high-fidelity front-end prototype of a patient management dashboard designed for endoscopy suites. This project was developed during my internship to solve the specific workflow challenges faced in procedural medical environments.

## Project Focus
The primary goal of this project was to design an interface that is both **highly readable** and **operationally efficient**. 

**Key UX/UI Highlights:**
- **"Slide-to-Start" Workflow:** An interactive interaction pattern designed to prevent accidental procedure initiation.
- **Clinical Status Visualization:** A color-coded status system (Complete/Pending/Incomplete/Bin) designed for quick "at-a-glance" status updates.
- **Dotted-Texture Branding:** A custom visual language designed to feel professional, sterile, and trustworthy.
- **Balanced Data Hierarchy:** Information-dense views (like the expanded patient record) optimized for clarity, ensuring the most important clinical information is always accessible.

## Project Status
**Note:** This is a **UI/UX Prototype (Front-end only)**.
- Data persistence and backend API integration are not implemented.
- This project focuses on visual design, interaction patterns, and responsive layout structure.
- All patient records shown in the live preview are mock data for demonstration purposes.

## Tech Stack
- **Design/Build:** HTML5, Tailwind CSS
- **Interaction Logic:** Vanilla JavaScript
- **Visuals:** Custom CSS for clinical textures and glow effects.
